⚙️ 6-Step Backend Execution Sequence

This is the exact logical flow the Java Spring Boot API enforces when a user hits "Submit" on a new expense. It must run inside an Atomic @Transactional block, meaning failure in any step mathematically rolls back the entire operation.

1 Validate Math First

Ensure total_amount > 0 and mathematically verify that all individual user splits sum up perfectly to the whole expense amount before touching the database.

2 Save Main Expense

Write the core expense to the database via INSERT INTO expenses (explicitly setting status = 'PENDING') and retrieve the generated Unique ID.

3 Log The Exact Splits

Loop through the subgroup members or custom users, running INSERT INTO expense_splits for each person, linking it exactly to the newly generated Expense ID.

4 Trigger 24-Hr Objections

Instantly spawn actionable tickets for all users (except whoever paid) by setting status to PENDING. If no objection in 24h, it auto-approves. If ANY objection blocks it, the backend instantly rejects.

5 Verify Category Budgets

The backend summarizes ONLY APPROVED expenses logged to this specific category. If it exceeds the group limits, the backend automatically runs INSERT INTO budget_alerts.

6 Append Immutable Audit Log

To finalize the transaction, the backend runs INSERT INTO history_log tracking exactly who initiated the expense request along with the JSON data. This guarantees zero-tampering traceability.
